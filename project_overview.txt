There was an initial version of the animation script (pf2eanimation-heuristic-fallback.js) that had a thorough vision for the project but that made some incorrect assumptions and didn't work correctly. In debugging. a programmer in interaction with an AI oversimplified the code to a functional script that lost most of the original vision (working_version_lacking_some_content.js).

The project goal is to take the functional foundation of the new bare-bones script and incorporate the ideas of the earlier script while retaining rock-solid engineering and bug-free execution. In particular, we want to retain a clear, coherent compatibility with existing Foundry scripts and the way the Sequencer database describes and accesses the JB2A animation library. The relevant scripts and library access codes are stored for convenience in the resources folder of the project. That folder also contains a copy of a GitHub repository containing the details of all PF2e spells; see resources/Pathfinder-2E-Spell-DB-master/SpellDB/src/data/spells.json

Our goals are to:

1) Reincorporate as many of the ideas from the original script as possible without breaking functionality.

2) Provide flavorful, thematically appropriate, and distinctive automated animation selection for PF2E spells, generating as distinctive and appropriate an experience for each spell as possible.

3) Map from the PF2E spell library onto as wide a range of the JB2A animations as feasible.

4) Avoid bugs, runtime errors, and other fragilities.

5) Maintain a high-quality runtime experience that does not put undo burden on the client or the machines running the program.
